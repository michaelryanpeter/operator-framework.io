What is an Operator?

Operators put operational knowledge into software. An Operator is a method of packaging, deploying, and managing a Kubernetes application. Operators automate common operations, such as installation, configuration, update, and backup. These activities run inside your Kubernetes cluster and integrate with Kubernetes APIs. We call this a Kubernetes-native application. A Kubernetes-native application is more than a collection of Pods, Deployments, and ConfigMaps. Operators are a single object that only exposes the parts that make sense for the application.

Who builds Operators?

Cluster Admins, Site Reliability Engineers, Software Engineers, and DevOps Engineers put their shared operational knowledge into an Operator. Their efforts create a resilient and robust solution that manages an application in a native Kubernetes way. Operators are available everywhere that OLM is installed.

Who uses Operators?

Cluster Admins extend the Kubernetes API by installing an Operator. Kubernetes users with the appropriate RBAC can interact with the Operator's APIs. End Users can interact with the applications installed by the Operator.

Capability model

  • level Ⅰ

    Basic install

    Automated application provisioning and configuration management

  • Level Ⅱ

    Seamless upgrades

    Patch and minor version upgrades supported

  • Level Ⅲ

    Full lifecycle

    App lifecycle, storage lifecycle (backup, failure recovery)

  • Level Ⅳ

    Deep insights

    Metrics, alerts, log processing and workload analysis

  • Level Ⅴ

    Autopilot

    Horizontal and vertical scaling, automatic configuration tuning, abnormal detection, scheduling tuning

Operators come in different maturity levels based on their lifecycle management capabilities for the application or workload they deliver. The capability model provides consistent terminology to help users understand what features they can expect from an Operator.
